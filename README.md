# Port Tunnel

Port Tunnel - прототип сервиса обратного TCP-туннелирования для публикации локального TCP-порта через сервер с публичным IP-адресом.

Проект позволяет открыть доступ к локальному приложению без настройки NAT и ручного проброса портов на маршрутизаторе.

Подробное описание компонентов, протокола и полного сценария взаимодействия находится в [ARCHITECTURE.md](ARCHITECTURE.md).

## Поддерживаемые сценарии

- несколько одновременных TCP-туннелей;
- публикация локального HTTP-сервера;
- проброс произвольного TCP-сервиса;
- WebSocket и другие протоколы поверх TCP.

UDP и авторизация в текущей версии не поддерживаются.

## Локальная проверка

Сервер:

```bash
cd src/server
python -m port_tunnel_server --control-port 7000 --public-host 127.0.0.1
```

Первый локальный сервис:

```bash
python -m http.server 8080
```

Первый клиент:

```bash
cd src/client
python -m port_tunnel_client \
  --server-host 127.0.0.1 \
  --control-port 7000 \
  --local-port 8080 \
  --public-port 30001
```

Проверка:

```bash
curl http://127.0.0.1:30001
```

Для второго туннеля запустите другой локальный сервис и клиент с другой парой портов, например `8081` и `30002`.
