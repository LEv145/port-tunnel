# Port Tunnel

Port Tunnel — прототип сервиса обратного TCP-туннелирования для публикации локального TCP-порта через сервер с публичным IP-адресом.

Проект позволяет открыть доступ к локальному приложению, например HTTP-серверу на `localhost:8080`, без настройки NAT и ручного проброса портов на маршрутизаторе.

## Поддерживаемые сценарии

- проброс локального HTTP-сервера;
- проброс произвольного TCP-сервиса;
- демонстрация локального web-приложения через публичный VPS.

UDP в текущей версии не поддерживается.

## Архитектура

Клиент устанавливает исходящее control-соединение с публичным сервером. Когда внешний пользователь подключается к публичному порту сервера, сервер отправляет клиенту команду открыть data-соединение. После этого сервер связывает внешнее подключение и data-соединение клиента, а клиент перенаправляет трафик на локальный порт.

## Локальный запуск

Терминал 1:

```bash
uv run src/server/port_tunner_server/app.py --control-port 7000 --public-port 30001
```

Терминал 2:

```bash
python -m http.server 8080
```

Терминал 3:

```bash
uv run src/client/port_tunner_client/app.py --server-host 127.0.0.1 --control-port 7000 --local-port 8080
```

Проверка:

```bash
curl http://127.0.0.1:30001
```
